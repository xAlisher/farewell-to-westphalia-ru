# Руководство для рецензентов

> **Книга:** Прощай, Вестфалия — Джаррад Хоуп и Питер Лудлоу
> **Задача:** Литературная вычитка русского перевода перед записью аудиокниги

---

## Как устроен процесс

Каждая глава — отдельный Pull Request. Ваша работа происходит целиком в GitHub.

```
Перевод готов → PR открыт → все трое рецензируют → автор правит →
@lead апрувит → мерж → автоматически готовится сценарий для диктора
```

Вам не нужно ничего устанавливать. Только браузер и аккаунт GitHub.

---

## Как оставить комментарий к строке

1. Откройте PR → вкладка **Files changed**
2. Наведите мышь на нужную строку → нажмите синюю **+** слева
3. Напишите комментарий → **Start a review** (не Add single comment — иначе каждый комментарий шлёт уведомление)
4. Когда закончили главу → **Finish your review** → выберите:
   - **Comment** — замечания есть, но апрув не ваша роль
   - **Request changes** — есть принципиальные ошибки, нужна правка
   - **Approve** — только для @lead

## Метки комментариев

Начинайте каждый комментарий с метки — это поможет автору расставить приоритеты.

| Метка | Когда использовать | Пример |
|---|---|---|
| `[ОШИБКА]` | Смысл оригинала искажён | `[ОШИБКА] «governance» здесь = контроль над протоколом, не «управление государством»` |
| `[СТИЛЬ]` | Фраза режет слух, но смысл верный | `[СТИЛЬ] «данный» звучит канцелярски, лучше «этот»` |
| `[ТЕРМИН]` | Сомнение по терминологии | `[ТЕРМИН]  «ДАО» или «DAО»? В тексте оба варианта` |
| `[АУДИО]` | Фраза сложна для произношения | `[АУДИО] Три слова подряд на -ции — диктор споткнётся` |
| `[ВОПРОС]` | Не уверены — задаёте вопрос автору | `[ВОПРОС] Авторы имеют в виду Вестфальский мир 1648 или принцип вестфальского суверенитета?` |

## На что обращать внимание

### Точность
- Смысл аргумента передан верно?
- Технические термины (блокчейн, смарт-контракт, ДАО) используются точно?
- Цитаты из третьих источников не искажены?

### Естественность русского языка
- Текст читается как русский, а не дословный перевод?
- Нет ли калек с английского? («в порядке» вместо «правильно», «делать смысл» вместо «иметь смысл»)
- Порядок слов естественный для русского?

### Синонимическая ротация
Скрипт автоматически варьирует повторяющиеся слова. Проверьте, что замены уместны:

- **управление / государственное устройство / форма правления** — подходит ли замена в этом конкретном предложении?
- **децентрализованный / распределённый / горизонтальный** — не меняет ли замена технический смысл?
- **Предварительные замечания / Вводные положения / К постановке вопроса** — отражает ли заголовок суть раздела?

Если замена неудачная — оставьте `[ТЕРМИН]` с предложением вернуть оригинал или выбрать другой вариант.

### Аудиокнига
Текст будет читать живой диктор в студии. Обращайте внимание на:
- Скопление шипящих или свистящих: «существующих систем» — тяжело
- Нанизывание падежей: «применения принципа управления развитием» — неудобно
- Очень длинные предложения без естественной паузы

---

## Чего не нужно делать

- **Не переписывайте целые абзацы** — это задача автора по вашим пометкам
- **Не спорьте о стиле друг с другом в комментариях** — пишите своё мнение, автор решает
- **Не апрувайте** — апрув делает только @lead после того, как автор обработал все замечания
- **Не трогайте строки в `[СКОБКАХ]`** — это режиссёрские ремарки для диктора, не часть текста

## Статусы PR

| Статус | Что происходит |
|---|---|
| 🟡 Open | Идёт рецензирование |
| 🔴 Changes requested | Автор правит по замечаниям |
| 🟢 Approved | @lead одобрил, можно мержить |
| ✅ Merged | Автоматически запущен финальный пасс |
